MemPalace — Semantic Project Memory

mempalace.yaml · 13 rooms
MemPalace is a vector knowledge graph MCP server. It maps project directories to "rooms" inside a "wing". Every architectural decision, past session, code pattern, and lesson learned is stored as a searchable drawer. The AI searches it before reading files for any design/debug question.
How it integrates with AI workflow
1. Main Claude always calls mempalace_search() before reading files for any design/debug/planning question
2. Subagents receive MemPalace results injected in their prompts — they cannot call it themselves
3. Session decisions, bugs, and patterns are written via mempalace_diary_write()
4. Knowledge graph queries via mempalace_kg_query(entity=…) for architectural decisions
When to use MemPalace vs Grep
MemPalaceSemantic/fuzzy questions, past decisions, "why did we…", "have we tried…", architectural context
Grep/ReadExact code/function names, structural navigation, editing files, finding specific patterns
Skip MPIf MCP unavailable — skip silently, grep session handoffs instead
