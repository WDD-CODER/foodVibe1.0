## What Went Wrong

### 1. Team Leader ghost-reporting recurred (same bug, session 2)
**What happened**: The Team Leader agent again claimed to write ~20 files when most were already committed. The same investigation pattern was needed: `git status` → sparse result → manual diff checks.

**Root cause**: The fix for this (Post-Delegation Diff Gate in `execute-it.md`) was implemented *in this session*, not before it. The Team Leader ran *before* the fix existed. This is expected — but it confirms the fix was necessary.

**Impact**: Low this session (pattern already familiar) — but would have been medium for any fresh agent.

### 2. Skill edit target had a mismatch for `execute-it`
**What happened**: The `/execute-it` skill content lives in `.claude/commands/execute-it.md` (a command file), not in a dedicated `skills/execute-it/SKILL.md`. The retrospective suggested editing "the execute-it skill" — required a Glob + Read to locate it.

**Root cause**: The project has both a `commands/` directory and a `skills/` directory; `execute-it` is a command, not a skill. The retrospective's "suggested file path" was ambiguous.

**Impact**: Minor — 2 extra tool calls to find the file. No wrong edits.

### 3. `subagent-driven-development` edit is in plugin cache
**What happened**: The implementer-prompt.md is at `plugins/cache/.../superpowers/5.0.7/...` — a version-pinned cache. The edit was made, but it may be overwritten on plugin update.

**Root cause**: No local override mechanism exists for superpowers plugin prompts in this project.

**Impact**: Low now, but the fix is fragile. If the plugin bumps to 5.0.8, the git-diff reporting rule will be lost.
