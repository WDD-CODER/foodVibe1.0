Type switch — how the yield row changes

Switching between מנה (dish) and הכנה (preparation) affects the yield field

מנה — Dish
מנות: מנות
Unit locked to "מנות" For a dish, yield = number of portions. The unit is always "מנות" — no dropdown needed. Maps to serving_portions in the builder.
הכנה — Preparation
תפוקה:
Unit is a free dropdown For a preparation, yield = weight/volume amount. Any unit from the app's registry can be chosen, plus "add new". Maps to yield_conversions[0] in the builder.
Note: There is no "minimum yield" field in the current data model — the builder doesn't have one either. The min=0 vs min=1 validation is just about whether 0 is a valid amount (preparations can have 0, dishes need at least 1 portion). If a "minimum batch size" field is a separate future requirement, that would need a data model change.
