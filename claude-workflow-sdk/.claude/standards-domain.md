# Domain Standards

> **[PLACEHOLDER]** This file is intentionally empty. After running `/init-repo`, fill in
> your project's domain-specific standards. Common sections to add:
>
> - **Terminology** — canonical names for domain entities (e.g. "product" vs "item" vs "record")
> - **Canonical value resolution** — how to resolve user input to canonical keys (e.g. units, categories)
> - **UI text / i18n** — translation key conventions, dictionary file location
> - **Icon registration** — where icons must be registered before use in templates
> - **Entity resolution flows** — what happens when a required canonical value is missing
>
> Load this file when: adding domain entities, handling canonical values, or working on domain logic.
